2021 年轉職前端工程師上工後，過了四年後的 murmur

![位於上野的東京都美術館](https://cdn-images-1.medium.com/max/800/1*bPFeg0H6b6SV85ZAIZf35g.jpeg)

自從找到工作後，就一直想紀錄「文組轉職工程師」後的故事，關於如何轉職網路上有無數篇教學和 roadmap，但實際上轉職後的故事卻甚少有文章分享。既上一篇文章是六角直播班心得，這次直接快轉四年，回顧我的短暫職涯。

---

### 背景

標準的非本科轉職工程師，日文系畢業，2021年時在家自學前端所需的語言、框架等知識，幸運地在年中趕上轉職潮紅利上岸，獲得第一份工作，也不知不覺待四年。

回顧當初，雖然準備了作品集，但後來問同事（當初的面試官）其實也沒有看。也不是技術特別強的人，只有照本宣科+自己做作品勤練習，離實戰還差得遠，但應徵職位也偏向 junior，剛好我那時的程度符合職位標準，才幸運找不到一個月就上岸。

但如果當初的程度，會切版、JS、只有自己做過 React+redux，但完全不懂什麼是 clean code、SOLID，拉到現在應該會被大洗臉一波吧…

### 現況

中型公司做了四年的前端開發，起初負責小範圍 issue & bug，後來接觸到功能設計、開發、寫測試等，從小專案的從頭開發到多框架的大型專案維護等都有實戰經驗，在這份工作中接觸到很多不同專案也奠基前端工程師應該要有的扎實基本功。

幸運地進入氛圍不錯的團隊，從一開始沒有實務經驗時有 mentor，後期接觸複雜專案架構時也有 senior 從旁協助，工作時間彈性，有需要也可以申請遠端在家。

讓我也對於工作型態有更多想像，可以不受地點時間限制，了解自己的喜好和 comfort zone。也意外發現其實自己很喜歡進辦公室有同事的感覺，沒有那麼喜歡 full remote 的自由，Hybrid 反而剛剛好，保持適當的距離和彈性。

### 真實工程師日常

#### 技術工作

想象中的工程師應該整天都在螢幕前敲鍵盤吧？

實際上可以**專心寫 code 的時間**佔整天工時比例 30%~20%，一天工作 8 小時寫 code 只有 2~3 小時，純粹個人經驗分享，如果是 senior 以上程度很可能除了吃飯時間以外，都被塞滿會議。

其他時間呢？

開會、討論專案、寫文件、code review、回覆訊息等意外地佔據不少時間，當這些事情佔據你大部分的工作時間後，逐漸覺得單純寫 code 寫邏輯真的好快樂…

「持續學習」也是工程師必須面對的一門課題，持續磨練不夠利的技術，學習不同的框架、語言、工具，也會和同事辦讀書會共讀。

這題也讓我想到**「資訊焦慮」**，如果不頻繁關注社群，很容易會因為深怕錯過社群趨勢，而感到焦慮，不停刷著社群儲存文章。同時，無力感油然而生，儘管下班總是疲憊，無法打起精神再學習。真的很好奇社群上的大神是如何持續保持動力？

![近期的我（不能逃！！！）](https://cdn-images-1.medium.com/max/800/1*k-MSYNvYsKjIV5ipe5nwHg.jpeg)

學著冥想、運動，重新找回身體的專注主控權是我近期的課題。

#### 技術之外

技術之外的溝通和釐清整理問題的能力。

正因為工作有一半以上比例不是寫扣，是由會議、文件撰寫、需求釐清等佔據，更需要溝通與釐清問題的能力。甚至我個人覺得在技術達到標準線的情境下，溝通遠比技術重要很多。

舉反例來說，看過身邊不善於溝通的案例（包括我），反覆來回確認需求，結果最後開發完成到測試，才發現和 PM 預想中有落差，不僅心理內耗，也浪費不少時間。

每當聽到同事精準溝通和釐清對話的過程，才發覺到自己多麼冗言贅字，雖然都是用中文溝通，卻發覺到自己詞彙量太少，總是無法準確表達意思。

#### 預料之外

意外發現喜歡/不喜歡的部分

![](https://cdn-images-1.medium.com/max/800/1*In0d5pM4OenPYWm_IynQpA.jpeg)

**Like**

- 喜歡單純撰寫功能邏輯，因為有明確目標不會太大改變
- 喜歡這份工作可以讓我在家工作，在喜歡的時區內工作，不會被制式上下班限制
- 意外喜歡不斷有新東西出現，可能偏奴（？）一成不變的環境會讓我懷疑人生
- 工作以外的時間有餘裕可以發展興趣

**Dislike**

- 溝通的比例意外的高，很多時候容易精神耗弱
- 在家工作不一定代表 WLB，有時會因為不想中斷心流而一路持續到半夜
- 「學不動了」的感覺隨著年紀越來越強烈

### 後悔嗎？

「你覺得你除了工作以外還有什麼？」離職後朋友問了我

剛開始工作的幾年，因為冒牌者症候群作祟，下班、休假日我總是在追技術，看開發文件，深怕自己跟不上拖累團隊。直到最近一兩年，習慣手頭上會接觸到的工作挑戰，也有自己應對壓力的方式，逼著自己下班後散步運動，減少資訊焦慮，才慢慢開始有餘裕去想更未來的事情。

離開工作後的現在，我總忍不住想回去以前的工作，總覺得寫扣手感在逐漸消失，會不會敏感度也消失？彷彿被空虛感淹沒的我，對於這個問題我什麼也答不出來。

是因為工程師才這樣嗎？（別瞎掰好嗎）

![](https://cdn-images-1.medium.com/max/800/0*W3zAqMV_0dvyY3gg)

不後悔當初的職涯選擇，儘管在 AI 浪潮下，他是受惠者，同時也是首當其衝的被取代者，但至少這技能帶著我去了很多地方看見更多可能性（g0v / daodaoedu …）

沒有一輩子都持續 programming 的自信，在這之外我還有什麼？是我之後想要探索的課題，目前還沒有可以抬頭挺胸說出的答案，期望有天會找到。

### 給 2026 還想轉職的你

如果看了以上勸世文，你還有轉職的想法

**現實面的建議**

- 培訓班相較之前少了很多，意指市場無法供給這麼多 junior 生存，是否可以接受找工作可能沒想像中容易
- 實務上與 AI 協同開發也會拉高對工程師產出的標準
- 雖然上面沒提到，剛入行工程師薪水可能沒你想像中的高，天花板高，但需要累積經驗後跳槽才可能疊高薪資
- 如果是抱著不想做「和人有關」的工作入行，那溝通佔比可能會讓你失望，需要更精準有效的溝通

如果看完網路上所有的經驗分享，還是想轉職的話，那就去嘗試吧！

頂多花費一年的時間發現不適合，再轉換跑道也不遲，學到腦裡的東西不會跑，說不定在人生的某個場合會再度派上用場，更何況現在是人人都在 vibe coding 的時代。

### 結語

也是說給自己聽的話

四年的總結心得來說，前份工作奠基我會未來公司、工作的理想標準，怎樣的工作型態是自己喜歡的、怎樣的產品開發模式相對舒服、理想工作的追求、role model 等，腦中有了雛形，工作判斷標準也明確許多，相比較剛畢業時不懂自己要什麼好太多。

希望在這行業還可多待上幾年，可以持續保有二十多歲的動力和好奇。未來規劃上，希望有機會開啟海外職涯的可能性，現在也持續尋找台灣 / 日本的前端工程師機會，雖然過程沒有想像中順利，但還是期望可以盡快落腳！回來報佳音！
